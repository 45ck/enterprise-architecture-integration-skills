# Pack Index

## Skills
- `enterprise-context-mapper` — Map the enterprise context around a system including business units, upstream/downstream systems, actors, governance constraints, and external dependencies.
- `enterprise-domain-alignment-checker` — Check whether a proposed solution aligns with enterprise business, application, information, and infrastructure domains rather than optimizing only one layer.
- `application-landscape-mapper` — Describe the surrounding application landscape so ownership, overlap, integration points, and replacement candidates are explicit.
- `soa-suitability-reviewer` — Assess whether service-oriented patterns are justified for the system based on autonomy, reuse, integration needs, and operational cost.
- `service-contract-designer` — Define service contracts, responsibilities, data ownership, and interface expectations for enterprise integrations.
- `loose-coupling-checker` — Review a design for coupling risks across services, teams, schemas, deployments, and runtime behavior.
- `message-bus-selector` — Choose an appropriate message bus or event backbone pattern based on integration scale, latency, delivery guarantees, and governance.
- `publish-subscribe-designer` — Design publish-subscribe interactions with explicit producers, consumers, events, topics, and failure handling.
- `rpc-vs-message-pattern-selector` — Decide whether synchronous RPC, asynchronous messaging, or a hybrid pattern best fits the integration context.
- `esb-fit-reviewer` — Evaluate whether an enterprise service bus is a fit, an anti-fit, or an overreach for the system and organization.
- `orchestration-vs-choreography-reviewer` — Review whether orchestration, choreography, or a mixed coordination model best suits cross-system workflows.
- `environment-topology-planner` — Plan enterprise environment topology across dev, test, staging, UAT, production, support, and disaster recovery needs.
- `container-vs-vm-advisor` — Advise when containers, virtual machines, or a mixed platform model are the safer enterprise choice.
- `ci-cd-readiness-checker` — Check whether architecture, environments, governance, and testing are ready for reliable CI/CD and controlled releases.
- `public-private-hybrid-cloud-selector` — Select between public, private, and hybrid cloud options with explicit attention to compliance, integration, latency, and operational realities.
